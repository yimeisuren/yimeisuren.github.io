debug的使用

命令 适用 功能
r 寄存器 查看寄存器值
r 寄存器 修改寄存器值
d 内存 查看内存值
e 内存 修改内存值
u 内存 将内存中原有的机器码翻译成所对应的汇编指令
T 内存 将CS:IP所指向的内存地址中的数据当作指令来执行
A 内存 以汇编指令的形式在内存中写入机器指令R

r命令

功能
查看和修改寄存器的值

语法

  • r+<寄存器名>

汇编语言中不能直接修改cs和ip的值, 而在debug中可以通过r命令直接修改cs和ip的值

d命令

功能
查看内存中的值

语法
d <段地址>[:<起始偏移地址>]

  • d + <段地址>
  • d + <段地址> + : +<起始偏移地址>
  • d + <段地址> + : + <起始偏移地址> + <结尾偏移地址>

结尾偏移地址可选字段

e命令

功能
修改内存中的内容

语法

  • e <起始地址> <数据1> [<数据2> ...] 连续修改内存中的值
  • e 段地址:偏移地址 逐个修改内存中的值, 结束输入敲enter, 继续输入敲空格
  • e 段地址:偏移地址 机器码 使用e命令向内存中写入机器码

u命令

功能

将内存中数据翻译成所对应的汇编指令

语法

  • u + <段地址> + : + <偏移地址>

a命令

功能

以汇编指令的形式向内存中写入数据

语法

  • a + <段地址> + : + <偏移地址>

t命令

功能

将CS:IP所指向的内存地址中的数据当作指令来执行

语法

  • t

t命令不带参数, 因为规定了执行cs:ip指向的内存地址, 只能去修改cs和ip的值


从图中可以看出, 执行t命令后, ax的值变为1, 即mov ax, 01这条汇编指令得到了执行

hexo搭建个人博客

下载nodejs使用npm

全局安装hexo

全局安装的作用是以后没有网络的时候也能建立工程

  1. 全局安装淘宝的cnpm, 方便加速

  2. 全局安装hexo

hexo使用流程

  1. 在IDEA中新建工程blogtest

  2. 打开命令行终端, 进入resources文件夹, 使用hexo init命令建立hexo项目. 必须为空文件夹才能用来初始化hexo项目

  1. 使用hexo n命令新建文章, 存放在生成的source->_posts文件夹下

  2. 使用Typora软件对文章进行修改后保存

  3. 使用hexo g命令对_post文件夹下的文章进行处理生成html文件, 以便放到网站上展示

  4. 使用hexo s命令启动项目进行本地预览

  5. 在github上新建仓库, 必须将其命名为: <用户名>.github.io (不需要尖括号“<”)

  6. 使用cnpm install --save hexo-deployer-git命令下载部署插件

  7. 修改_config.yml配置文件

  1. 使用hexo d命令推送到远端

  2. 输入网址 https://yimeisuren.github.io 对个人博客进行访问

修改博客主题

  • 网上搜索hexo主题,找到github的地址, 并克隆到blog文件夹下. 下载慢速的情况下加入.cnpmjs.org后缀

    其中themes/next为blog文件夹下的子文件夹

  • 修改配置文件_config.yml

  2. 在hexo d推送之前, 要先下载部署器

Typora配置图床

创建仓库

  1. 在gitee或github上创建任意一个仓库, 用来存放将要上传的图片
  2. 创建仓库的过程中注意要开源, 否则其他人无法访问

获取私人令牌

获取私人令牌, 以便Picgo能够以持有者的身份操作仓库

配置Picgo程序

  1. 在插件设置中搜索gitee, 下载gitee插件, 点击安装即可
    不要下载gitee-uploader插件, 个人尝试两个插件均下载后会产生冲突, 导致错误

  2. 在图床设置->Gitee图床中, 显示如下内容

    仓库的url链接为: https://gitee.com/andewkarsa/pictures

    • owner: andrewkarsa 复制粘贴防止出错
    • repo: pictures

注意: 可能由于Picgo的版本不同导致字段的含义有所差别, 具体可参考C:\Users\Administrator\AppData\Roaming\picgo\node_modules\picgo-plugin-gitee-uploader\Readme.md文件中解释

Typora配置Picgo

  1. 文件->偏好设置->图像
  2. 按上图配置好后, 出现验证成功则表示成功实现图床的配置
  3. 注意: 图片名重复也会导致上传失败, 可以在picgo.log文件中查询具体出错信息
